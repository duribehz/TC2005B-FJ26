Preguntas del Laboratorio 2

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Evitarlo, ya que es una mala practica porque ataca los fundamentos de diseño de CSS, porque anula el sistema natural de CSS, y si se comparte el codigo con otras personas seria muy dificil trabajar sobre el, dificultando el debugging y el mantenimiento.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Porque elegir una mala imagen afecta rendimiento, legibilidad, accesibilidad y experiencia del usuario en general. Hablando en el diseño se debe elegir también cuidadosamente una imagen que coincida con la estatica de la página.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt

Aunque en la pregunta no la menciona la mejor opción para usar seria REM o EM, pero de las que menciona la pregunta pues ya depende mucho de lo que se quiere hacer, por encima recomendaria la unidades relativas antes que las fijas o las unidades de impresión, pero depende de que se quiere hacer.

  • px (Unidad absoluta): Una de sus ventajas es que es muy precisa y fácil de controlar, pero no es responsivo y si se quiere usar la página en otros dispositivos rompe toda la estetica.
  • % (Unidad relativa): Esta unidad es responsiva entonces si el sitio esta diseñado para otros dispositivos esta va muy bien, su uso puede ser algo confuso ya que el % es relativo a su elemento padre.
  • pt (Unidad de impresión): Esta casi no se usa en web moderno, solo si se quiere para documentos PDF o de impresión. No es responsivo y no es nada apropiada para pantallas.
¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

La versión minimizada del CSS ya se volvio una practica que toda la industria practica y las razones por la cual mejora el rendimiento son varias.

  • Reduce el tamaño por lo cual tiene menos bytes que descargar.
  • Descarga más rapida por lo cual el usuario tiene un tiempo de espera mas bajo.
  • Renderiza más rapido porqe el navegador procesa más rápido.
