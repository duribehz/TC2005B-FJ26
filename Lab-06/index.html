¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Para detectar errores antes de que se realice la petición y mejorar la experiencia de usuario. Se reduce el tráfico innecesario y evitando que el servidor pierda tiempo procesando datos que no son correctos, pero nunca debe sustituir a la validación en el backend.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Puedes desactivar JS del navegador, con las herramientas de desarrollador puedes manipular el DOM o también y haciendo un post directo al servidor

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque una cosa es la experiencia de usuario y otro tema es la seguridad del sistema. Si tengo un error de dedo pues es mejor que me marque error antes de enviarlo al server, ya las validaciones del backend es la verdadera defensa del sitio.
