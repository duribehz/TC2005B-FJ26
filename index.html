¿Cuáles son las partes de un URL?
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: Se utiliza para obtener y consultar datos del servidor.
  • HEAD: Este es parecido al GET, pero las respuestas tienen encabezados y cuerpos, la diferencia con el GET es que el HEAD solo devuelve devuelve los encabezados.
  • POST: Para enviar datos al servidor.
  • PUT: Se usa para actualizar o reemplazar un recurso y si el recurso no existe lo crea.
  • PATCH: Tambien funciona para actualizar un recurso pero no por completo, solo parcialmente.
  • DELETE: Para eliminar un recurso.
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método POST porque envia la información dentro del cuerpo del menaje HTTP.
"Los datos enviados a través de POST no son visibles en la URL. Si se utilizara GET para una contraseña, esta quedaría expuesta en el historial del navegador, en los registros del servidor (logs) y para cualquier persona que mire la pantalla del usuario" (Mozilla MDN, 2023).

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

EL método utilizado es el GET, porque este sirve para pedir la representeción de un recurso. "El método HTTP GET solicita una representación de un recurso específico. Las peticiones que utilizan GET solo deben recuperar datos" (Mozilla MDN, 2023).

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Cuando un servidor web devuelve un código 200, significa que todo bien, significa que todo OK, recibio y proceso correctamente la solicitud del navegador. No ocurrio ningún error todo salio exitosamente.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Tecnicamente no ya que un código 404 es un error cliente, significa que no puede encontrar el recurso solicitado, pero la causa puede ser provocada por un fallo en la gestión del sitio.

"Un error 404 no indica si el recurso falta de forma temporal o permanente. Sin embargo, los enlaces que conducen a un error 404 suelen conocerse como 'enlaces rotos' o 'muertos' y están sujetos a la pudrición del enlace (link rot)" (Mozilla MDN, 2023).

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si es responsabilidad del desarrollador. El código 500 nos dice que el servidor encontró una condición inesperada que no lo dejo completar la solicitud. "El código de estado 500 indica que el servidor encontró una condición inesperada que le impidió cumplir con la solicitud. Es la respuesta por defecto cuando el servidor no puede encontrar un código de error 5xx más específico" (IETF, 2014).

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Aunque los navegadores suelen seguir soportándolos por compatibilidad con sitios antiguos, no deben usarse en desarrollos nuevos" (W3C, 2021).

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML 4 usaba muchísimo la etiqueta div y con los identificadores id, para definir la estructura de la página, es decir en ves en lugar de usar la etiqueta header, usaban una div id = "header". HTML 5 usa las etiquetas semánticas, que son las etiquetas que usamos ahora como el header, nav, footer, etc.

"La semántica de HTML5 permite que las máquinas comprendan la estructura del contenido, mejorando la accesibilidad y el SEO al diferenciar claramente el encabezado, el pie de página y el contenido principal" (W3C, 2014).

En HTML 4 para poder utilizar audio o un video utilizabas un servicio externo y HTML5 ya tiene sus etiquetas de audio y video lo cual lo hace más practico.HTML 5 mejoró también mucho la parte de gráficos dinámicos ya que HTML 4 era muy limitado en esa parte.

HTML 5 tiene otras opciones de almacenamiento, Web Storage(localStorage y sessionStorage), esto permite guardar 5MB o 10MB de datos en el cliente sin afectar el rendimiento de la red, a diferencia de HMTL 4 que relentizaba la carga utilizaba principalmente las cookies para guardar datos en el navegador y estaban limitados a 4KB y se enviaban con cada petición HTTP.

Para declarar una archivo HTML 4 utilzaba una referencia enorme a un DTD, mientras que HTML 5 solo usa una linea.

Una de las mejores cosas de HTML 5 es su soporte al conjunto de APIs.
"HTML5 no es solo una actualización del lenguaje, sino un conjunto de tecnologías que incluyen CSS3 y nuevas APIs de JavaScript que permiten crear aplicaciones web ricas que compiten con las aplicaciones nativas" (Mozilla MDN, 2023).

¿Qué componentes de estructura y estilo tiene una tabla?

"Para estructurar una tabla de forma profesional, debes utilizar una jerarquía semántica en HTML que separe el contenido en tres secciones principales: el encabezado (), el cuerpo de datos () y el pie de tabla (). Dentro de estas, las filas se definen con , diferenciando claramente las celdas de título () de las celdas de datos comunes (), lo cual es fundamental para la accesibilidad y el SEO (W3C, 2023)."
En cuanto al estilo, la presentación se gestiona mediante CSS para transformar la estructura básica en una interfaz legible. Los componentes esenciales incluyen la propiedad border-collapse para unificar los bordes, el uso de padding para dar aire al texto y la aplicación de selectores como :nth-child para crear filas de colores alternos (efecto cebra), lo que facilita el seguimiento visual de la información (Mozilla MDN, 2023).

¿Cuáles son los principales controles de una forma HTML5?
  • Campos de texto: (input) Es la clasica cajita para ingresar texto, pero se puede modificar su atributo type para tener diferentes funcionalidades, como validar que solo se ingresen correo electronicos o para no mostrar el texto cuando ingresas una password, validar que solo se esta ingresando texto, etc.
  • Selección de opciones: Existen los radios buttons para elegir sola una opción de varias y los checkboxes para elegir varias opciones.
  • Listas despegables: Es para elegir un elementos o varios de una lista, en lugar de una uno de seleccion, con las listas despegables se ahorra espacion en pantalla.
  • Controles de fecha y Color: Despliegan calendarios o paletas. Son selectores nativos de HTML5
  • Áreas de texto extenso: Son como los campos de texto, solo que se utilizan cuando se necesita un texto extenso.
  • Botones de acción: Se puede cambiar su atributo type depende la función que necesitemos, existe el submit que funciona para ennviar un formulario o el reset para limpiar algún campo.

Sobre el ciclo de vida y desarrollo de los sistemas de información:
¿Cuál es el ciclo de vida de los sistemas de información?

Según Laudon & Laudon (2021), las etapas principales son:

  1. Nacimiento y Planificación: Identificación de una necesidad o problema organizacional y determinación de la viabilidad.
  2. Desarrollo: Creación técnica del sistema (aquí es donde entra el ciclo de desarrollo).
  3. Operación: El sistema está en uso productivo procesando transacciones y generando informes.
  4. Mantenimiento: Corrección de errores y adaptación a nuevos requerimientos.
  5. Muerte o Eliminación: El sistema se vuelve obsoleto y es reemplazado o desactivado.
¿Cuál es el ciclo de desarrollo de sistemas de información?

De acuerdo con Kendall & Kendall (2014), el ciclo de desarrollo estándar (modelo en cascada) consta de las siguientes fases:

  1. Identificación de problemas y oportunidades: Entender qué debe resolver el software.
  2. Determinación de los requerimientos de información: Entrevistas y recolección de datos con los usuarios finales.
  3. Análisis de las necesidades del sistema: Creación de diagramas de flujo de datos y diccionarios de datos.
  4. Diseño del sistema: Diseño de la interfaz, la base de datos y la arquitectura lógica.
  5. Desarrollo y documentación del software: Fase de programación (codificación).
  6. Pruebas y mantenimiento del sistema: Verificación de que el código no tenga errores.
  7. Implementación y evaluación: Instalación del sistema y capacitación a los usuarios.
